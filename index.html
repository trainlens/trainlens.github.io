Architecture

Lightweight. Out-of-process. Zero interference.

TrainLens runs a separate aggregator process alongside your training loop. No CUDA stream contention. No process group hijacking.

Wrap your training step with trace_step(), use TrainLensTrainer, or attach TrainLensCallback. Works with any PyTorch codebase.

Quickstart

Up and running in two minutes.

from trainlens.decorators import trace_step

for batch in dataloader:
    with trace_step(model):
        outputs = model(batch["x"])
        loss    = criterion(outputs, batch["y"])
        loss.backward()
        optimizer.step()
        optimizer.zero_grad(set_to_none=True)

    # optional: auto-terminate on predicted failure
    if check_terminate_signal(session_dir):
        break
$ trainlens run train.py 
from trainlens.integrations.huggingface import TrainLensTrainer

trainer = TrainLensTrainer(
    model=model,
    args=training_args,
    train_dataset=train_ds,
    eval_dataset=eval_ds,
    trainlens_enabled=True,
)
trainer.train()

# drop-in replacement for Trainer
# pip install "trainlens-ai[hf]"
import lightning as L
from trainlens.integrations.lightning import TrainLensCallback

trainer = L.Trainer(
    max_epochs=10,
    callbacks=[TrainLensCallback()],
)
trainer.fit(model, datamodule)

# standard Lightning callback
# pip install "trainlens-ai[lightning]"
# Live bottleneck diagnosis
$ trainlens run train.py

# Per-layer timing + memory
$ trainlens deep train.py

# Gradient diagnostics (MFU, NaN/Inf, FSDP)
$ trainlens run train.py --grad-diagnostics

# Multi-GPU DDP
$ trainlens run train.py --nproc-per-node=4

# Review run history
$ trainlens history

Use Cases & Whitepaper

From single-GPU experiments to org-wide training infrastructure.

TrainLens operates at every scale — a solo researcher debugging a single GPU, a team managing a cluster, or an organization tracking training health across hundreds of concurrent runs.

Researcher
Debug a slow training run in minutes

You notice step time doubled between two runs. TrainLens shows the dataloader wait spiked from 2ms to 180ms — a dataset preprocessing bottleneck that nvidia-smi wouldn't reveal.

ML Team
Find the rank slowing your DDP job

An 8-GPU DDP run is 30% slower than expected. TrainLens shows rank 3 is consistently 40ms behind the median — a single slow NVLink link causing the whole job to wait at the barrier.

MLOps
Stop failing runs before they waste GPU hours

A long training job starts showing signs of gradient instability at step 800. TrainLens predicts failure and terminates the run cleanly — saving 6 hours of A100 time before the inevitable crash.
